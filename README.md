## Настройка ОС
В нашей команде все (в основном) работают на Mac OS. Mac OS построена на Unix платформе, так же как и Linux, если есть опыт работы в этих системах, то разобраться не составит труда (Ubuntu почти один-в-один). Если опыт работы был только с Windows, то придется немного привыкнуть, но об этом попозже.

Так-как до тебя на этой машине уже кто-то работал, то скорее всего осталось много ненужного хлама. Если хочешь можно оставить как есть, но я бы порекомендовал сделать полный сброс до заводских настроек.
### Сброс до заводских настроек
Сброс в Mac OS делается очень просто, но требует установки системы с нуля, а это значит полная очистка диска и установка системы через интернет, что может занять в среднем от получаса до полутора часов в зависимости от скорости интернета (у нас интернет быстрый).
1. Убедиться в наличии интернета
2.  → Перезагрузить
3. В процессе перезагрузки следует зажать на клавиатуре `⌘Cmd + R`, после чего Mac загрузится в режиме восстановления, в котором необходимо будет выбрать язык (русский присутствует).
4. Затем появится окно **«Утилиты macOS»**. Здесь нужно выбрать **«Дисковая утилита»** и кликнуть **«Продолжить»**
5. Выбрав слева загрузочный диск (обычно это **Macintosh HD**), перейдите в раздел **«Стереть»** (в верхней части экрана)
6. В меню **«Формат»** выберите **APFS** и нажмите **«Стереть»**
7. По завершении процесса форматирования (сброса в заводские настройки) нажмите **«Завершить»** для выхода из Дисковой утилиты
8. Переустановите macOS, воспользовавшись соответствующим пунктом и следуйте инструкции. Последняя версия macOS будет загружена из Интернета
9. В заключительных настройках нужно будет войти в Apple ID (спросить у кого нибудь доступы) и создать своего пользователя в системе.

### Немного про Mac OS для новичков
Есть постоянная панель управления которая всегда находится вверху экрана. В ней расположены основной меню , меню активного приложения и трей. Через основное меню  можно перейти в настройки мака, а также Выключить/Перезагрузить машину (аналогично кнопке Пуск в винде). Если есть желание настроить машину под себя, можно открыть **Системные настройки...** и пройтись по всем пунктам.

Также имеется бар с быстрым доступом к программам (стандартно он находится внизу экрана, можно в настройках изменить положение на экране). Туда можно добавить наиболее часто используемые программы (редактор, FTP клиент, браузеры и т.д.) и убрать ненужные.

Очень полезная функция Spotlight, вызывается с помощью сочетания клавиш `Ctrl + Space`. Это умный быстрый поиск по вашей машине. Начните вводить название программы или имя файла и система предложит вам наиболее подходящие варианты. Если привыкнуть можно вообще перестать пользоваться баром с программами и проводником.

Файловая система отличается от Windows. Здесь нет привычной иконки с Моим компьютером и разделением на диски. Mac OS разделяет информацию на диске по пользователям. Когда вы создавали своего пользователя, на диске была создана специальная пользовательская директория, которая и является вашей основной директорией для работы с файлами.

Чтобы попасть в папку пользователя, нужно войти в проводник (в маке это приложение Finder, обычно всегда присутствует в баре с быстрым доступом), в левой колонке нажать Загрузки и перейти на директорию выше с помощью сочетания `⌘Cmd + ↑`, либо ввести в терминале команду `open ~` (см. [Основные команды для работы с терминалом](#Основные-команды-для-работы-с-терминалом)).

Я рекомендую создать в этой директории новую директорию *develop* где будут храниться все рабочие проекты. Так же можно перетащить эту директорую в левую колонку проводника Finder чтобы иметь быстрый доступ к ней.

## Настройка рабочих сервисов
### Почта
Это ваш основной аккаунт на который будут зарегистрированы некоторые рабочие сервисы. Также используется для внутреннего общения с клиентами и работниками компании.

Аккаунт будет зарегистрирован в формате *i.ivanov@pugofka.com*
### Битрикс24
Основная система для работы с задачами и учетом времени.

Основные моменты:
1. По приходу в офис не забываем нажать "Начать рабочий день"
2. При длительных перерывах (обед и т.д.) наживаем "Приостановить". Не забываем нажать "Продолжить" при возвращении.
3. Ваши задачи находятся в пункте **Задачи и проекты → Делаю**
4. Следим за дедлайнами и затраченным временем. Дедлайн включает в себя полное завершение задачи с тестированием и проверкой выполнения руководителем, менеджером и/или клиентом. Если выполнение задачи выходит за установленные сроки, нужно заранее предупредить об этом постановщика задачи, руководителя или менеджера в каком либо виде (в комментарии к задаче, словестно, в чат и т.д.)
5. Приступая к задаче не забываем нажать кнопку "Начать учет времени". Можно приостановить учет времени нажав кнопку "На паузу". Также время можно указать вручную во вкладке "Время".
6. Завершая задачу (см. пункт 4) нажимаем кнопку "Завершить".
7. По завершению рабочего дня нажимаем кнопку "Завершить рабочий день".
### GitLab
Локальный сервис для работы с Git. Находится на нашем dev сервере. Используется в основном для проектов в которых работаешь один. Доступы выдаются руководителем.
### BitBucket
Основной сервис Git для работы над закрытыми проектами в команде. Можно использовать уже имеющийся аккаунт, либо зарегистрироваться если аккаунта нет. Регистрироваться можно и на личную и на рабочую почту, как удобнее.
### GitHub
Основной сервис Git для работы над открытыми проектами. Можно использовать уже имеющийся аккаунт, либо зарегистрироваться если аккаунта нет. Регистрироваться можно и на личную и на рабочую почту, как удобнее.

## Установка ПО
### SourceTree
Настольная программа для работы с Git репозиториями. Рекомендуется к использованию во избежание косяков. [Скачать с официального сайта](https://www.sourcetreeapp.com/).
### AvoCode
Программа для удобной верстки макетов. Кушает файлы PhotoShop и Sketch. Доступы выдаются руководителем. [Скачать с официального сайта](https://avocode.com/download).
### FTP
Для работы вам понадобится FTP менеджер. Можно использовать любой удобный для вас клиент. Рекомендуемые клиенты: [Cyberduck](https://cyberduck.io/) и [FileZilla](https://filezilla-project.org/download.php?type=client).

## Установка среды разработки
### Node.js
Для работы с фронтендом вам понадобится среда Node.js. [https://nodejs.org/en/download/](Скачать с официального сайта). Также автоматически будет утановлен пакетный менеджер npm.
### n
Утилита n позволяет установить несколько версий Node.js и легко переключаться между ними. Для установки утилиты введите в терминале команду `sudo npm install -g n`. Более подробно об использовании утилиты можно [прочитать в официально репозитории](https://github.com/tj/n).
### Homebrew
Пакетный менджер Mac OS. Понадобится для установки других пакетов через терминал. Чтобы скачать и установить откройте терминал и выполните команду `sudo /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
`. Чтобы убедиться что утсановка выполнена успешно введите в терминале команду `brew -v` она должна показать текущую версию Homebrew.
### Yarn
Аналог пакетного менеджера npm, но быстрее и лучше. В рабочих проектах рекомендуется работать через него а не через npm. Для установки ввести в терминале команду `sudo brew install yarn`. Чтобы убедиться что утилита установлена ввести в терминале команду `yarn --version` она должна показать текущую версию Yarn.
### Gulp
Для работы с автоматическими сборками верстки мы используем утилиту Gulp. Для установки утилиты введите в терминале команду `sudo npm install gulp-cli -g`.  Чтобы убедиться что утилита установлена введите в терминале команду `gulp -v` она должна показать текущую версию Gulp.
### Docker
Для работы с проектами написанными на PHP (или другими серверными языками) вам понадобится среда для запуска таких проектов. Сейчас хорошей практикой считается использовать Docker. Докер позволяет создать виртуальное окружение абстрагируемое от основной системы для установки в ее переделах нужных вам инструментов разработки и версий. [Скачать с официального сайта](https://www.docker.com/community-edition).

## Установка и настройка редактора
Для разработки фронта сегодня удобнее всего использовать один из трех редаткоров: [Sublime Text](https://www.sublimetext.com/), [Atom](https://atom.io/) и [Visual Studio Code](https://code.visualstudio.com/). На данный момент самым оптимальным вариантом для разработки на фронте является [Visual Studio Code](https://code.visualstudio.com/) поскольку он довольно быстр (по сравнению c [Atom](https://atom.io/) написанном на той же технологии [Electron](https://electron.atom.io/)) и максимально заточен под разработку на фронте (встроенные функции дебагинга, GIT, IDE фичи для JS, терминал и т.д.). Все дальнейшие настройки будут касаться только [Visual Studio Code](https://code.visualstudio.com/), но с большой долей вероятности все тоже самое можно настроить и на вышеупомянутых редакторах, если вы будете их использовать.

### Базовые настройки
Для вызова окна настроек редактора нужно в левом баре нажать иконку шестеренки и вырать **Параметры** или нажать ⌘,

Настройки в VSCode выполнены в виде json файла. Для станадартной настройки я бы рекомендовал следующие параметры:
```json
{
  "workbench.colorTheme": "Monokai",
  "editor.renderWhitespace": "boundary",
  "editor.multiCursorModifier": "ctrlCmd",
  "editor.wordWrap": "on",
  "editor.tabSize": 2,
  "editor.formatOnSave": true,
  "editor.minimap.showSlider": "always",
  "editor.formatOnPaste": true,
  "editor.snippetSuggestions": "top"
}
```
Вы можете изменить эти и другие настройки редактора по своему усмотрению.

### Расширения
Для установки расширения нужно в левом баре выбрать **Расширения** или нажать ⇧⌘X и ввести в поиске название плагина.
#### ESLint
Первый плагин для достижения единого код-стайл в проектах. Линтеры это программы, которые в режиме реального времени, по заданным для него правилам, проверяют ваш код на ошибки.
После установки расширения вам также понадобится глобальная установка утилиты eslint в систему. Для этого в терминале нужно ввести команду `npm install -g eslint`.

Если в корне проекта присутствует файл `.eslintrc` то плагин будет ориентироваться на его правила. У популярных фреймворков вроде **[React](https://www.npmjs.com/package/eslint-plugin-react)** или **[Vue.js](https://github.com/vuejs/eslint-plugin-vue)** уже есть встроенные правила ESLint. Если же не используется фреймворк, я бы рекомендовал с помощью команды `eslint --init` автоматически составить правила выбрав из списка вариант **Airbnb**.

Подробнее про ESLint можно почитать на [официальном сайте](https://eslint.org/).

#### Prettier
Второй плагин на пути единого код-стайл. Данный плагин занимается тем что автоматически форматирует код в красивый читаемый вид.

#### Sublime Text Keymap
Если вы привыкли к горячим клавишам Sublime Text то этот плагин легко интегрирует все хоткеи из сайблайма в VSCode.

#### Babel ES6/ES7
Поддержка синтаксиса ES6/ES7 в редакторе.

#### Vetur
Поддержка синтаксиса Vue.js в редакторе.

#### Path Intellisense
Автокомплит путей до файлов.

#### Git Lens
Большое расширение для работы с гитом в редакторе.

#### Settings Sync
Удобный плагин для синхронизации своих настроек и зависимостей редактора. Вы можете в качестве примера загрузить мои настройки редактора добавив в праметры строчку `"sync.gist": "4f55df7d0a4e4198bc3005d27d463760"` и засинхронить параметры с помощью ⇧⌥D. Таким образом вы получите все вышеуказанные параметры и плагины + дополнительно все чем я пользуюсь в повседневной разаработке.

P.S. также я бы порекомендовал установить [Sublime Text](https://www.sublimetext.com/) поскольку он отлично подходит для быстрого редактирования одиночных файлов и в целом может быть неплохой заменой станадартному блокноту.
