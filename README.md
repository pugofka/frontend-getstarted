## Установка и настройка редактора
Для разработки фронта сегодня удобнее всего использовать один из трех редаткоров: [Sublime Text](https://www.sublimetext.com/), [Atom](https://atom.io/) и [Visual Studio Code](https://code.visualstudio.com/). На данный момент самым оптимальным вариантом для разработки на фронте является [Visual Studio Code](https://code.visualstudio.com/) поскольку он довольно быстр (по сравнению c [Atom](https://atom.io/) написанном на той же технологии [Electron](https://electron.atom.io/)) и максимально заточен под разработку на фронте (встроенные функции дебагинга, GIT, IDE фичи для JS, терминал и т.д.). Все дальнейшие настройки будут касаться только [Visual Studio Code](https://code.visualstudio.com/), но с большой долей вероятности все тоже самое можно настроить и на вышеупомянутых редакторах, если вы будете их использовать.

### Базовые настройки
Для вызова окна настроек редактора нужно в левом баре нажать иконку шестеренки и вырать **Параметры** или нажать ⌘,

Настройки в VSCode выполнены в виде json файла. Для станадартной настройки я бы рекомендовал следующие параметры:
```json
{
  "workbench.colorTheme": "Monokai",
  "editor.renderWhitespace": "boundary",
  "editor.multiCursorModifier": "ctrlCmd",
  "editor.wordWrap": "on",
  "editor.tabSize": 2,
  "editor.formatOnSave": true,
  "editor.minimap.showSlider": "always",
  "editor.formatOnPaste": true,
  "editor.snippetSuggestions": "top"
}
```
Вы можете изменить эти и другие настройки редактора по своему усмотрению.

### Расширения
Для установки расширения нужно в левом баре выбрать **Расширения** или нажать ⇧⌘X и ввести в поиске название плагина.
#### ESLint
Первый плагин для достижения единого код-стайл в проектах. Линтеры это программы, которые в режиме реального времени, по заданным для него правилам, проверяют ваш код на ошибки.
После установки расширения вам также понадобится глобальная установка утилиты eslint в систему. Для этого в терминале нужно ввести команду `npm install -g eslint`.

Если в корне проекта присутствует файл `.eslintrc` то плагин будет ориентироваться на его правила. У популярных фреймворков вроде **[React](https://www.npmjs.com/package/eslint-plugin-react)** или **[Vue.js](https://github.com/vuejs/eslint-plugin-vue)** уже есть встроенные правила ESLint. Если же не используется фреймворк, я бы рекомендовал с помощью команды `eslint --init` автоматически составить правила выбрав из списка вариант **Airbnb**.

Подробнее про ESLint можно почитать на [официальном сайте](https://eslint.org/).

#### Prettier
Второй плагин на пути единого код-стайл. Данный плагин занимается тем что автоматически форматирует код в красивый читаемый вид.

#### Sublime Text Keymap
Если вы привыкли к горячим клавишам Sublime Text то этот плагин легко интегрирует все хоткеи из сайблайма в VSCode.

#### Babel ES6/ES7
Поддержка синтаксиса ES6/ES7 в редакторе.

#### Vetur
Поддержка синтаксиса Vue.js в редакторе.

#### Path Intellisense
Автокомплит путей до файлов.

#### Git Lens
Большое расширение для работы с гитом в редакторе.

#### Settings Sync
Удобный плагин для синхронизации своих настроек и зависимостей редактора. Вы можете в качестве примера загрузить мои настройки редактора добавив в праметры строчку `"sync.gist": "4f55df7d0a4e4198bc3005d27d463760"` и засинхронить параметры с помощью ⇧⌥D. Таким образом вы получите все вышеуказанные параметры и плагины + дополнительно все чем я пользуюсь в повседневной разаработке.

P.S. также я бы порекомендовал установить [Sublime Text](https://www.sublimetext.com/) поскольку он отлично подходит для быстрого редактирования одиночных файлов и в целом может быть неплохой заменой станадартному блокноту.
